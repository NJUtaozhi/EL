## 二、已完成工作详情

### 2.1 LLM 服务 — [services/llmService.ts](services/llmService.ts)

封装 DeepSeek API 调用，为第3周归因分析做准备：

| 方法 | 说明 |
|------|------|
| `chatCompletion(options)` | 通用多轮对话，支持自定义 temperature/maxTokens/responseFormat |
| `singlePrompt(systemPrompt, userPrompt, options?)` | 单轮快捷调用（System + User → Response） |

**特性**：
- ✅ **自动重试** — 5xx 错误最多重试 2 次，4xx 不重试（如 401 认证失败）
- ✅ **超时控制** — 30 秒超时
- ✅ **结构化日志** — 每次 API 调用记录 attempt 次数和 token 消耗
- ✅ **返回 token 用量** — promptTokens / completionTokens / totalTokens

**调用示例**：
```typescript
const result = await chatCompletion({
  messages: [{ role: 'user', content: '你好' }],
  temperature: 0.7,
  maxTokens: 1024,
})
// → { content: "...", usage: { promptTokens: 10, completionTokens: 50, totalTokens: 60 } }
```

### 2.2 任务服务 — [services/taskService.ts](services/taskService.ts)

| 方法 | 说明 |
|------|------|
| `createTask(userId, dto)` | 创建任务，自动补全 weather + semesterPhase |
| `getTasks(userId, options?)` | 查询任务列表（按日期范围 + limit 筛选），含关联 analysis |
| `getTaskById(userId, taskId)` | 查询单个任务详情（含归因结果） |
| `deleteTask(userId, taskId)` | 删除任务（校验所有者），级联删除关联 analysis |

**自动补全逻辑**：
- **学期阶段** — 根据当前月份判断：1-2月→假期 / 3-5月→平时 / 6-7月初→考试周 / 7-8月→假期 / 9-12月→平时
- **天气** — 根据季节简单推断（后续可接入真实天气 API）

### 2.3 任务控制器 — [controllers/taskController.ts](controllers/taskController.ts)

| 方法 | 路由 | 说明 |
|------|------|------|
| `createTask` | POST /api/tasks | 校验 body → 调用 service 创建 → 返回 201 |
| `getTasks` | GET /api/tasks | 支持 query 参数 startDate/endDate/limit |
| `getTaskById` | GET /api/tasks/:id | 参数校验 + 404 处理 |
| `deleteTask` | DELETE /api/tasks/:id | 所有者校验 + 404 处理 |

### 2.4 任务路由 — [routes/task.ts](routes/task.ts)

```
POST   /api/tasks       ← 创建任务（auth + Zod 校验）
GET    /api/tasks       ← 任务列表（auth）
GET    /api/tasks/:id   ← 任务详情（auth）
DELETE /api/tasks/:id   ← 删除任务（auth）
```

**Zod 校验规则**：
- `title` — 必填，1-200 字
- `reason` — 可选，最长 500 字
- `emotion` — 可选，枚举 `开心 | 焦虑 | 平静 | 沮丧 | 生气`

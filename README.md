# EL
比赛相关规则：

赛采用组队形式参赛，每队1～4人。大赛设置交互组、算法组、AI智能体创新专项组三大赛道。三个赛道各有侧重，赛道独立、成果衔接，无论你是偏好自由探索的创意表达者，还是擅长逻辑推理的技术挑战者，或是希望借助AI工具快速实现想法的实践派，都能在EL的舞台上找到属于自己的发光点。


一、交互组
交互组为开放式创意赛道，注重作品的创意表达与落地能力。


在这里，参赛者不需要拘泥于固定命题，可以从身边的生活细节出发，也可以聚焦社会议题与时事热点，结合自身学科背景与兴趣特长，借助AI、大语言模型等技术手段，将优秀的idea与扎实的工程实现相结合，实现创意落地。


以下方向仅供参考，参赛团队可自主拓展选题：
· 智慧校园类：课程助手、学习规划、校园服务、活动管理、信息聚合等；
· 智慧生活类：健康管理、时间管理、兴趣社交、生活工具、智能提醒等；
· AI应用类：基于大语言模型的问答助手、资料整理、内容生成、智能推荐等；
· 绿色低碳类：环保监测、碳排放动态模型、排污权交易辅助系统等；
· 行业场景类：智能运维平台、数据分析工具、企业管理系统、软硬件协同应用等。


参赛对象


南京大学2025级本科生


核心赛制


1. 参赛团队自主组队、自主选题，围绕真实需求完成作品设计与开发。
2. 6月上旬开展中期检查，通过提交的项目文档或PPT评估项目思路成熟度，若未通过中期检查的组别将无法参与最终评比。
3. 暑假开始前作品提交截止。参赛团队需提供项目文档或PPT、演示视频、项目源码仓库链接。专家评委组将对参赛团队提供的项目文档与演示视频打分，按获奖队伍数1:1.5的比例确定下一轮晋级队伍。
4. 暑假开始前进行线下答辩。最终将根据评委评分确定比赛名次。


特别说明：交互组赛道作品可进一步打磨，直接报名参与AI智能体创新专项组比拼，一次创作、持续升级、两次冲奖！

三、AI智能体创新专项组
AI智能体创新专项组为EL大赛×火山引擎联合设立的校企合作专属赛道，以低门槛、高落地、强赋能为特色，面向全校学生开放AI应用创新场景，让创意快速变成可运行的智能体。
在该赛道中，依托火山引擎官方赛事平台，提供完整开发环境、云资源、模型能力与培训支持，零代码/低代码即可开发，技术基础薄弱也能轻松上手。


一次参赛、双重收获！既可参与EL大赛评奖，优秀作品更有机会被推荐至火山杯Agent创新大赛全国赛，在更大的平台上展示创意、交流经验、链接资源。


参赛对象


南京大学全体在校生（不限年级、不限专业）


核心赛制


1. 赛道主题
聚焦智慧校园、智慧生活两大方向，围绕学习、生活、服务、管理、安全、效率等真实场景，用 AI 智能体解决实际问题，鼓励积极向上、对校园与社会有益的创新应用。
2. 开发工具与平台
统一使用火山引擎官方赛事管理平台报名、提交、评审；
需使用火山引擎・扣子（Coze）作为智能体开发工具；
平台提供在线开发环境、大模型能力、产品代金券、官方教程与实验沙箱。
3. 参赛路径
交互组团队：可在原有作品基础上升级改造为 AI 智能体参赛；
非交互组同学/团队：直接报名。
4. 赛程安排
7月：中期检查（检查开发进度与方向合规性，未通过取消参评资格）；
8月：完整作品提交（含智能体体验链接/部署包、项目文档、演示视频、源码/配置说明）；
9月：线下答辩评审，综合创新价值、场景落地性、体验完整度、技术合规性打分。
5. 评审与晋级
校内评审：评选一/二/三等奖；
校外推荐：优秀作品由校方推荐至火山杯Agent创新大赛，参与全国评选，进一步拓展项目展示、交流学习和资源对接平台。

我们的选题：
🚀「不拖延实验室」升级版 —— 拖延溯源分析 + 智能干预系统
一、升级定位
从“归因分析”升级为「诊断 → 干预 → 追踪」闭环拖延管理系统。
不再只是告诉你“为什么拖延”，而是真正帮你减少拖延，并量化改变过程。
二、核心升级点（对比原版）
维度	原版	升级版
输入方式	手动填写任务+拖延理由	支持文字、语音、多轮对话（像跟朋友吐槽一样）
归因能力	简单规则/轻量模型归类	Coze 工作流 + 知识库：深度追问，识别深层原因（如恐惧失败、完美主义）
输出内容	拖延画像+建议	个性化干预方案+ 微习惯打卡 + 每日提醒
追踪机制	无	情绪与拖延趋势曲线，展示干预效果
落地形式	单纯分析	小程序/Web + 智能体双引擎，可嵌入微信/飞书
双赛道适配	无	交互组：完整前端+后端；专项组：Coze 智能体驱动
三、交互组方案（第一轮参赛）
产品形态
•	微信小程序 或 响应式 Web 应用
•	风格：简洁、治愈、数据可视化
核心功能模块
1. 拖延日志（核心输入）
•	快速记录：今日任务 + 拖延理由（可语音转文字）
•	自动补全时间、天气、学期阶段（考试周/平时）
2. 智能归因引擎（轻量版）
•	前端调用 OpenAI / 本地 LLM API（或先期用规则+关键词）
•	输出归类：畏难型 / 焦虑型 / 贪玩型 / 无规划型 / 完美主义型
•	生成“拖延标签云”
3. 拖延画像 + 可视化
•	周/月报告：拖延频率、主要类型分布
•	热力图：一周哪天最易拖延
•	曲线图：干预前后对比
4. 简单干预建议
•	针对不同归因，推送预设建议卡片（如“分解任务”“5分钟启动法”）
•	可收藏建议，生成每日“防拖提醒”
5. 打卡与微习惯
•	每天完成1个“最小行动”（如写50字、收拾桌面）
•	连续打卡奖励虚拟勋章
技术栈建议（交互组）
•	前端：Uniapp / Taro（跨端）或 React + TDesign
•	后端：Node.js / Flask + MySQL
•	模型：初期使用 API（DeepSeek / 智谱），中期可本地部署轻量分类模型
•	可视化：ECharts
交付物
•	项目文档 + PPT
•	演示视频（3分钟）
•	源码仓库 + 小程序体验版二维码
四、专项组升级方案（第二轮）
升级策略：保留前端，将核心归因与干预逻辑全部迁移至火山引擎 Coze 智能体
1. 智能体架构设计
用户输入（文字/语音） 
   ↓
Coze 智能体（主 Agent）
   ├─ 工作流1：深度归因
   │   ├─ 调用 LLM 进行多轮追问（“你当时是什么感觉？”“开始做的话会担心什么？”）
   │   ├─ 检索知识库（心理学拖延分类文献）
   │   └─ 输出：归因类型 + 置信度
   ├─ 工作流2：干预方案生成
   │   ├─ 根据归因类型，从知识库匹配干预策略（如“拆分任务”“番茄钟”“认知重构”）
   │   ├─ 若用户情绪低落 → 加入安抚话术 + 白噪音链接
   │   └─ 输出：可执行的3步行动建议
   ├─ 数据库记录
   │   ├─ 每次交互时间、归因结果、任务名称
   │   └─ 用于生成历史趋势图
   └─ 插件调用（可选）
        ├─ 日历插件：为用户生成明日计划
        └─ 消息推送插件：每日提醒打卡
2. 关键 Coze 组件使用
组件	用途
知识库	存放拖延心理学模型、干预策略、鼓励语录、校园心理资源
工作流	归因分析 → 干预方案 → 风险判断（若严重则推咨询渠道）
数据库	存储用户历史记录（脱敏），支持趋势图表 API
多模态	语音输入（火山引擎 ASR） + 可选情绪识别（从语音/文本）
Bot 对话界面	嵌入原小程序/Web 中（Coze SDK 或 iframe）
3. 前后端改造
•	原前端调用 Coze 的 API（发送用户消息，接收智能体返回的归因+建议）
•	原可视化页面仍保留，数据从 Coze 数据库同步读取（或通过 Coze 开放 API）
•	新增“与 AI 对话”Tab，用户可直接与智能体多轮聊天，智能体自动记录
4. 专项组评审亮点
•	创新性：不只是工具，而是有记忆、会追问、能干预的心理式助手，区别于普通任务管理 App
•	技术实现：Coze 工作流 + 知识库 + 数据库 + 多模态，完全符合“低门槛、高落地”
•	实用价值：直接缓解大学生最大痛点，数据证明干预效果（可展示曲线下降）
•	交互体验：对话自然，可视化清晰，支持语音输入，降低使用门槛

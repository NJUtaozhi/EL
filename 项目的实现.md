# 「不拖延实验室」—— 交互组项目实现文档

> **团队人数**：2 人  
> **当前阶段**：交互组（第一轮）  
> **技术栈**：React + TDesign + ECharts + Node.js(Express) + MySQL(Prisma) + **DeepSeek API**

---

## 一、当前架构（交互组）

```
┌──────────────────────────────────────────────────┐
│                   用户入口                         │
│           响应式 Web 应用 (React)                   │
└─────────────────────┬────────────────────────────┘
                      │  REST API
          ┌───────────▼───────────┐
          │   后端 (Node.js)       │
          │   Express API          │
          │   ┌─────────────────┐  │
          │   │  DeepSeek API   │  │
          │   │  (归因分析+建议)  │  │
          │   └─────────────────┘  │
          └───────────┬───────────┘
                      │
          ┌───────────▼───────────┐
          │   MySQL 数据库          │
          └───────────────────────┘
```

**说明**：交互组阶段，前端 React 直接调用后端 API，后端封装 DeepSeek 实现归因分析和干预建议生成。

> ⚠️ Chat 页面（AI 对话）在交互组阶段为**轻量实现**——直接调后端 → DeepSeek API 做简单多轮对话。专项组阶段再迁移到 Coze 工作流。

## 五、DeepSeek API 调用说明

### 5.1 配置（`.env`）

```env
LLM_API_KEY=sk-your-deepseek-key
LLM_API_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

### 5.2 归因分析调用（`analysisService.ts` 核心逻辑）

```
输入：用户任务名 + 拖延理由 + 情绪标签
  ↓
组装 Message:
  System: "你是拖延心理学专家。根据用户描述判断拖延类型：
          1.畏难型 2.焦虑型 3.贪玩型 4.无规划型 5.完美主义型
          返回JSON: {type, confidence, keywords, suggestion}"
  User:   "任务：写期末论文。理由：不知道怎么开头，一拖再拖。情绪：焦虑"
  ↓
调 DeepSeek API → 解析返回 JSON
  ↓
存入数据库 Analysis 表 → 返回给前端展示
```

### 5.3 干预建议调用（`interventionService.ts`）

```
输入：归因类型 + 用户历史
  ↓
System Prompt 包含各类型的干预策略库（拆分任务/番茄钟/认知重构等）
  ↓
DeepSeek 生成 3 步行动建议 + 鼓励语
  ↓
返回建议卡片列表
```

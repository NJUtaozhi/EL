展示 DeepSeek AI 返回的归因分析结果：

- **类型头部**：大图标 + 类型名 + 描述文字 + 置信度百分比
- **关键词区**：圆角标签，按类型配色

```tsx
// 使用方式

```

卡片以 `scaleIn` 弹入动画呈现，边框颜色与类型匹配。

### 2.3 Analysis 归因分析页面 `pages/Analysis/index.tsx` 🔴 必做

完整的归因分析页，支持四种状态无缝切换：

#### 状态一：选择任务
- **快速创建**：输入框 + "分析"按钮，一键创建任务并触发分析
- **已有任务列表**：从 zustand store 读取，点击触发分析
- **空引导**：无任务时提示去记录页添加
- 支持 URL 参数 `?taskId=123`（从 Log 页跳转预选任务）

#### 状态二：分析中
- 旋转加载动画（CSS spinner）
- "DeepSeek AI 正在解读你的拖延模式"
- 显示正在分析的任务名

#### 状态三：结果展示
- 归因结果卡片（AttributionResult）
- 拖延关键词云（TagCloud）
- 操作按钮：分析其他任务 / 查看干预建议

#### 状态四：错误处理
- 友好错误图标 + 提示信息
- 重试按钮

#### 数据流

```
任务选择 → submitForAnalysis(taskId) → DeepSeek后端归因
                                      ↓
                              解析JSON返回
                                      ↓
                   AttributionResult ← 类型+置信度+关键词+建议
                   TagCloud        ← 关键词可视化
```

## 五、技术要点

### 5.1 状态机设计

使用 `PageState` 枚举管理页面四种状态（`'select' | 'analyzing' | 'result' | 'error'`），每种状态对应独立的 UI 渲染分支，逻辑清晰、无状态交叉。

### 5.2 路由参数联动

支持 URL 查询参数 `?taskId=123`，Log 页面点击任务卡片后跳转至此，Analysis 页自动选中任务并触发分析，实现无缝页面过渡。

### 5.3 词云内存管理

TagCloud 组件使用 `useRef` 持有 ECharts 实例，在组件卸载时通过 `useEffect` 的 cleanup 函数调用 `dispose()` 释放资源，避免内存泄漏。

### 5.4 快速创建快捷路径

无需先切到 Log 页创建任务，可在 Analysis 页直接输入任务名 → 一键创建 → 立即触发分析，减少页面跳转。

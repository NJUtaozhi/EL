## 1. 作品介绍

### 1.1 项目背景

拖延是当代大学生普遍面临的心理行为问题。调查显示，超过 70% 的大学生存在不同程度的学业拖延行为，严重影响学习效率和心理健康。然而，大多数学生并不清楚自己拖延的深层原因，也缺乏科学有效的应对方法。

### 1.2 项目概述

**不拖延实验室（Anti-Procrastination Lab）** 是一款面向大学生的**拖延诊断与智能干预系统**。项目以"认知—行为疗法（CBT）"为理论基础，构建了 **诊断 → 归因 → 干预 → 追踪** 的完整闭环：

```
记录拖延行为 → DeepSeek AI 归因分析（5型分类）
  → 可视化拖延画像（饼图 / 热力图 / 趋势图）
  → 个性化干预方案（CBT 三步法）
  → 每日微行动打卡 + 徽章激励系统
```

### 1.3 核心功能

| 功能模块 | 说明 |
|---------|------|
| **拖延记录** | 记录每次拖延行为，支持语音输入、情绪标签、学期阶段标记 |
| **AI 归因分析** | 调用 DeepSeek API，将拖延行为自动归类为 5 种类型，附带置信度和关键词 |
| **拖延画像** | ECharts 多图表可视化：环形饼图（类型分布）、双折线趋势图、热力图（星期×时段） |
| **干预方案** | AI 生成个性化三步行动方案，内置 5 种预设策略库，支持收藏 |
| **每日打卡** | 微行动建议 + 一键打卡 + 月历视图，建立正向习惯 |
| **徽章系统** | 7 枚徽章自动判定与发放，进度追踪，获得动效通知 |
| **用户系统** | 自动登录（设备指纹）、头像上传、昵称编辑、数据持久化 |

## 2. 作品目标

### 2.1 核心目标

1. **帮助用户认知拖延**：通过 AI 归因分析，让用户从"我知道我在拖延"升级为"我理解我为什么拖延"，实现从模糊感知到精准分类的认知跃迁。

2. **提供科学干预**：基于认知行为疗法（CBT），为每种拖延类型提供个性化的三步行动方案，将抽象的"别拖延了"转化为具体可执行的行动指令。

3. **建立正向循环**：通过每日微行动打卡 + 徽章激励系统，帮助用户用微小的成功体验替代拖延的愧疚感，逐步建立自我效能感。

4. **可视化成长轨迹**：多维度数据可视化（饼图、折线图、热力图、词云），让用户直观看到自己的拖延模式和改善趋势。

### 2.2 技术目标

- 构建完整的全栈 Web 应用（React + Express + MySQL + AI）
- API 响应时间 < 100ms（不含 AI 调用）
- AI 归因分析准确率 > 80%（基于 Prompt 工程优化）
- 移动端优先的响应式设计，3 档适配（手机 / 平板 / 桌面）
- 代码分割 + 懒加载，首屏 JS < 200KB (gzip)
- 50 个后端测试用例，核心 API 覆盖率 > 90%

### 2.3 用户价值

- **认知层面**：从"我不知道为什么拖延"到清楚自己的拖延类型和触发模式
- **行为层面**：从"知道要改但不知道怎么改"到拥有可执行的个性化行动方案
- **情感层面**：从"我是不是没救了"的自我否定到"原来这是可以改变的"的自我效能感

#### 环境要求

#### 快速部署（推荐）

**Step 1**：克隆代码

```bash
git clone https://github.com/NJUtaozhi/EL.git
cd EL
```

**Step 2**：配置环境变量

```bash
# 后端环境变量（已有默认值，可直接使用）
# backend/.env
DATABASE_URL="mysql://root:el123456@localhost:3307/el_db"
JWT_SECRET="el_jwt_secret_2026"
DEEPSEEK_API_KEY="your_deepseek_api_key"
DEEPSEEK_BASE_URL="https://api.deepseek.com"
PORT=3000
```

**Step 3**：构建

```bash
# 后端：安装依赖 + 编译 TypeScript
cd backend
npm install
npm run build

# 前端：安装依赖 + 打包
cd ../frontend
npm install
npm run build
```

**Step 4**：启动

```bash
# 回到项目根目录
cd ..
# 双击运行 start.bat
# 或在终端中执行：
./start.bat
```

`start.bat` 会自动完成：
1. ✅ 检查 Docker 运行状态
2. ✅ 启动 MySQL 容器（端口 3307）
3. ✅ 等待 MySQL 就绪
4. ✅ 执行 Prisma 数据库迁移
5. ✅ 启动后端服务（端口 3000）
6. ✅ 启动 Nginx 容器（端口 80）
7. ✅ 自动打开浏览器 → `http://localhost:80`

### 5.2 使用指南

#### 5.2.1 首次使用

1. 打开浏览器访问 `http://localhost:80`
2. 系统会自动分配设备 ID 并完成登录（无需手动注册）
3. 进入首页，查看仪表盘概览

#### 5.2.2 记录拖延行为

1. 点击底部 Tab「📝 记录」
2. 点击紫色「+ 记录一次拖延」按钮展开表单
3. 填写拖延内容（必填，最多 50 字）
4. 可选：输入拖延原因（支持**语音输入**，点击麦克风按钮）
5. 选择当时的情绪标签（5 种可选）
6. 点击提交，任务卡片出现在下方列表

#### 5.2.3 AI 归因分析

1. 点击底部 Tab「📊 分析」
2. 方式一：从已有任务列表中选择一个任务
3. 方式二：直接输入新的拖延行为描述，点击分析
4. 等待 3-5 秒，AI 返回结果：
   - **拖延类型**（5 型之一）+ 置信度百分比
   - **关键词标签**（彩色圆角标签）
   - **AI 建议文字**
   - **词云图**可视化关键词权重分布

#### 5.2.4 查看拖延画像

1. 点击底部 Tab「👤 我的」
2. **用户卡片**：点击头像可上传自定义图片（支持 JPG/PNG/GIF，最大 2MB），点击昵称可编辑（1-20 字）
3. **环形饼图**：查看 5 种拖延类型的分布占比
4. **双折线趋势图**：紫色线 = 本周每日拖延任务数，蓝色线 = 每日打卡数
5. **热力图**：横轴上午/下午/晚上 × 纵轴周日~周六，颜色深浅表示拖延频率

#### 5.2.5 获取干预方案

1. 点击底部 Tab「💡 干预」
2. 选择你想改善的拖延类型（点击类型卡片高亮）
3. 点击「AI 生成个性化方案」获取 DeepSeek 定制的三步行动方案
4. 点击 ⭐ 收藏喜欢的方案
5. 也无需等待 AI：内置 5 种预设策略可直接查看

#### 5.2.6 每日打卡

1. 点击底部 Tab「✅ 打卡」
2. 查看系统推荐的**微行动建议**（可点击「换一个」随机切换）
3. 点击中央大圆形按钮完成打卡（动效反馈：紫色 → 绿色 + 勾选弹跳）
4. 查看**打卡月历**（紫色圆圈标记已打卡日期）
5. 浏览**徽章墙**（7 枚徽章，已获得 = 金色发光，未获得 = 灰度 + 进度条）
6. 如解锁新徽章，会自动弹出庆祝动效

#### 5.2.7 徽章系统

| 徽章 | 图标 | 获得条件 |
|------|------|---------|
| 初次打卡 | 🎯 | 完成首次打卡 |
| 连续三天 | 🔥 | 连续打卡 3 天 |
| 坚持一周 | ⭐ | 连续打卡 7 天 |
| 两周达人 | 🏅 | 连续打卡 14 天 |
| 月度之星 | 👑 | 连续打卡 30 天 |
| 记录达人 | 📝 | 累计记录 10 条任务 |
| 自我认知 | 🧠 | 完成 5 次归因分析 |

徽章在以下时机自动判定：
- **打卡后**：检查打卡系列徽章（5 枚）
- **创建任务后**：检查记录达人徽章
- **完成分析后**：检查自我认知徽章

#### 5.2.8 用户数据管理

数据自动持久化到 MySQL 数据库，支持：
- 同一设备自动登录（localStorage 存储设备 ID）

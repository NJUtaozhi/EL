整个自动登录机制涉及三个文件：App.tsx、api/index.ts（拦截器）、api/user.ts、store/userStore.ts。

自动登录流程
第 1 步：生成设备指纹

// App.tsx
function getDeviceId(): string {
  const key = 'device_openid'
  let id = localStorage.getItem(key)
  if (!id) {
    // 首次访问：随机生成一个唯一 ID
    id = 'web_' + Math.random().toString(36).slice(2) + Date.now().toString(36)
    localStorage.setItem(key, id)  // 存起来，下次还用这个
  }
  return id
}
每个浏览器生成一个唯一的设备 ID，存在 localStorage 里永久不删。下次打开网站还是同一个身份。

第 2 步：每次打开都重新登录

// App.tsx - useEffect，仅在组件挂载时执行一次
useEffect(() => {
  const openId = getDeviceId()
  login(openId, '访客')     // 调后端登录接口
    .then((res) => {
      setToken(res.token)    // token 存 localStorage + zustand
      setUser(res.user)      // 用户信息存 zustand
    })
    .finally(() => {
      setAuthReady(true)     // 不管成功失败，都放行渲染页面
    })
}, [])
⚠️ 之前这里有个 bug：先检查 if (token) 有旧 token 就跳过。改掉之后每次都重新登录，保证 token 始终有效。

第 3 步：后端登录（注册/登录二合一）

POST /api/user/login  { openId: "web_abc123...", nickname: "访客" }
后端逻辑：


根据 openId 查数据库
  ├─ 找到了 → 更新昵称，返回已有用户
  └─ 没找到 → 创建新用户

返回 { user: { id, nickname, ... }, token: "eyJ..." }
同一个设备 ID 永远对应同一个用户，数据不会丢。

第 4 步：拦截器自动带 token

// api/index.ts - 请求拦截器
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`  // 每个请求自动带
  }
  return config
})

// 响应拦截器 - 自动解包
api.interceptors.response.use(
  (res) => {
    if (res.data.code === 0) return res.data.data  // 自动解包内层 data
    throw new Error(res.data.message)
  }
)
第 5 步：token 过期怎么办？
JWT 有效期是 7 天。但由于每次打开 App 都重新登录（第 2 步），每次拿到的都是全新 token，所以只要不是连续 7 天不关浏览器，token 永远不会过期。

完整链路图

浏览器打开 http://localhost:80
        │
        ▼
   App 组件挂载
        │
        ├─ getDeviceId() ─── localStorage 取/生成设备 ID
        │
        ├─ login(openId, '访客')
        │      │
        │      ▼
        │   POST /api/user/login ─── 后端查/建用户 ─── 返回 JWT token
        │      │
        │      ▼
        │   setToken(token) ─── localStorage.setItem('token', token)
        │   setUser(user)   ─── zustand store
        │   setAuthReady(true)
        │
        ▼
   渲染页面（Home / Log / Profile ...）
        │
        ▼
   页面调 API（如 getTasks()）
        │
        ├─ 请求拦截器：从 localStorage 取 token ──→ 加到 Authorization 头
        │
        ├─ 后端验证 JWT ──→ 返回数据
        │
        └─ 响应拦截器：code===0 ? 解包 data : 抛异常
              │
              ├─ 成功 → 页面展示数据
              └─ 失败 → 页面显示"加载失败" + 重试按钮
关键设计决策
决策	原因
设备 ID 存 localStorage 不删	保证同一浏览器始终是同一用户
每次打开都重新登录	旧 token 可能过期，每次都拿新的最安全
登录失败也渲染页面	后端挂了不应该白屏，让各页面自己显示重试按钮
openId 模式而非用户名密码	项目原本为微信小程序设计，Web 版用设备 ID 模拟

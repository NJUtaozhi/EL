### 2.6 用户模块（Routes → Controller → Service）

**路由**：[routes/user.ts](routes/user.ts)

| 方法 | 路径 | 说明 | 认证 | 校验 |
|------|------|------|------|------|
| POST | `/api/user/login` | 登录/注册（openId 模式） | 无 | Zod(body) |
| GET | `/api/user/me` | 获取当前用户信息 | Bearer token | 无 |

**控制器**：[controllers/userController.ts](controllers/userController.ts)

- `login` — 接收 openId + 可选昵称/头像，调用 service 层返回 user + token
- `getMe` — 从 `req.user` 提取 userId，返回用户信息

**服务层**：[services/userService.ts](services/userService.ts)

- `login(dto)` — openId 查找用户：存在则更新信息，不存在则创建；签发 JWT（7天有效期，604800s）
- `getUserById(userId)` — 按 ID 查询用户基本信息

## 四、架构说明

当前后端分层架构已搭建完毕：

```
请求 → 路由(routes) → 中间件(auth/validator) → 控制器(controllers) → 服务(services) → Prisma → MySQL
                                                    ↕
                                              模型层(models) 工具层(utils)
```

数据流：`routes/user.ts → middleware/validator.ts(zod校验) → controllers/userController.ts → services/userService.ts → prisma → MySQL`

错误流：任意层抛出异常 → `middleware/errorHandler.ts` 统一捕获 → `utils/response.ts` 格式化输出

## 二、已完成工作详情

### 2.1 归因分析服务 — [services/analysisService.ts](backend/src/services/analysisService.ts)

**核心方法**：

| 方法 | 说明 |
|------|------|
| `analyzeTask(userId, taskId, options?)` | ⛓️ 核心归因：取任务 → 组 Prompt → 调 DeepSeek → 解析 JSON → 存 Analysis 表 |
| `getAnalysisByTaskId(userId, taskId)` | 仅查数据库，不做 LLM 调用 |
| `getUserAnalyses(userId, options?)` | 用户分析历史（分页），含关联 task 信息 |
| `getTypeDistribution(userId)` | 拖延类型分布统计，用于饼图 |

**归因 Prompt 工程**（精心设计的 System Prompt）：

```
角色定位：拖延心理学专家，精通认知行为疗法和时间管理理论

输出要求：严格 JSON 格式
{
  "type": "畏难型",
  "confidence": 0.85,
  "keywords": ["太难了", "做不完"],
  "suggestion": "建议将任务拆分为..."
}

5种拖延类型的详细定义：
- 畏难型：任务大，怕失败 → 特征关键词：太难了、做不完、不会做
- 焦虑型：过度担忧结果 → 特征关键词：担心、紧张、怕出错
- 贪玩型：娱乐分散注意 → 特征关键词：玩手机、刷视频、打游戏
- 无规划型：缺乏时间管理 → 特征关键词：不知道先做什么、没计划
- 完美主义型：过度追求完美 → 特征关键词：还没准备好、不够完美
```

**JSON 解析容错**（`parseAnalysisJSON`）：

| 场景 | 处理方式 |
|------|----------|
| LLM 返回纯 JSON | 直接 `JSON.parse` ✅ |
| LLM 包裹 markdown 代码块 | 正则提取 ` ```json ... ``` ` 内的内容 |
| LLM 在文本中嵌入 JSON | 正则提取第一个 `{...}` 对象 |
| 缺少必填字段 | 提供安全默认值（confidence=0.5, keywords=[], suggestion="暂无建议"） |
| 数值越界 | 钳制 confidence 到 [0, 1]；keywords 最多 4 个；suggestion 最长 200 字 |

**幂等设计**：
- 已分析过的任务默认返回缓存（不重复消耗 token）
- 传入 `forceRefresh: true` 可强制重新分析并更新数据库
- 使用 Prisma `upsert` 保证不存在并发写入冲突

**User Prompt 组装**（`buildUserPrompt`）：
```
【任务名称】写期末论文
【拖延理由】不知道怎么开头，一拖再拖
【当前情绪】焦虑
【学期阶段】考试周
```

### 2.2 归因控制器 — [controllers/analysisController.ts](backend/src/controllers/analysisController.ts)

| 方法 | 路由 | 说明 |
|------|------|------|
| `submitAnalysis` | POST /api/analysis/submit | 提交归因分析（调 LLM），返回 201 |
| `getAnalysisResult` | GET /api/analysis/result/:taskId | 获取已存储的分析结果（只读） |
| `getHistory` | GET /api/analysis/history | 用户分析历史列表，支持 limit/offset 分页 |
| `getDistribution` | GET /api/analysis/distribution | 拖延类型分布 → `{ "畏难型": 5, "焦虑型": 2 }` |

**错误处理**：
- `taskId` 非数字 → 400 "无效的 taskId"
- 任务不存在/无权访问 → 404
- LLM JSON 解析失败 → 502 "AI 返回结果解析失败"
- DeepSeek API 异常 → 经 llmService 重试后仍然失败，透传至全局 errorHandler

### 2.3 分析路由 — [routes/analysis.ts](backend/src/routes/analysis.ts)

```
POST   /api/analysis/submit       ← 提交归因分析（auth + Zod 校验）
GET    /api/analysis/result/:taskId ← 获取分析结果（auth）
GET    /api/analysis/history       ← 分析历史列表（auth，支持分页）
GET    /api/analysis/distribution  ← 类型分布统计（auth）
```

**Zod 校验规则**：
- `taskId` — 必填，正整数
- `forceRefresh` — 可选，布尔值

## 三、核心数据流

```
用户在前端 Analysis 页面点击"提交分析"
  │
  ▼
POST /api/analysis/submit  { taskId: 3 }
  │
  ▼
routes/analysis.ts → auth 中间件（验证 JWT） → Zod 校验 body
  │
  ▼
controllers/analysisController.submitAnalysis()
  │
  ▼
services/analysisService.analyzeTask(userId, taskId)
  │
  ├─ 1. Prisma 查 task（含 analysis 关联）
  │     └─ task 不存在/不属于用户 → throw 404
  │
  ├─ 2. 缓存检查：task.analysis 存在且非 forceRefresh → 直接返回缓存 ✅
  │
  ├─ 3. buildUserPrompt(task) → 组装 User Prompt
  │     "【任务名称】写期末论文\n【拖延理由】不知道怎么开头..."
  │
  ├─ 4. llmService.singlePrompt(systemPrompt, userPrompt, {temperature:0.5, responseFormat:'json_object'})
  │     │
  │     ▼ DeepSeek API
  │     ┌─────────────────────────────────────────────────────┐
  │     │ POST https://api.deepseek.com/v1/chat/completions    │
  │     │ model: deepseek-chat                                 │
  │     │ temperature: 0.5                                     │
  │     │ response_format: { type: "json_object" }             │
  │     │ → 自动重试 2 次，30s 超时                             │
  │     └─────────────────────────────────────────────────────┘
  │     │
  │     ▼ 返回 JSON
  │     { "type": "焦虑型", "confidence": 0.88, "keywords": [...], "suggestion": "..." }
  │
  ├─ 5. parseAnalysisJSON(raw) → 容错解析（去 markdown/提取首个{}/默认值填充）
  │     └─ 解析失败 → throw 502 "AI 返回结果解析失败"
  │
  ├─ 6. prisma.analysis.upsert({ taskId }) → 存入数据库
  │     └─ 已存在则更新（幂等）
  │
  └─ 7. 返回 AttributionResult → 前端展示
```

额外提供的聚合接口：

```
GET /api/analysis/history?limit=20&offset=0
  → analysisService.getUserAnalyses(userId, {limit, offset})
    → Prisma 查 analysis 表（连 task），按时间倒序，分页
      → 返回 AnalysisWithTask[]（含任务标题、理由等）

GET /api/analysis/distribution
  → analysisService.getTypeDistribution(userId)
    → Prisma 按 type 分组计数
      → 返回 { "畏难型": 5, "焦虑型": 2, "贪玩型": 1, ... }
        → 前端直接绑定 ECharts 饼图
```

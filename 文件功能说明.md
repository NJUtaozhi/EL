# 「不拖延实验室」项目文件功能说明

> **团队人数**：2 人  
> **技术栈**：React + TDesign + ECharts + Node.js(Express) + MySQL(Prisma) + DeepSeek API  
> **架构**：前端 SPA → REST API → 后端 Express → DeepSeek AI / MySQL  
> **开发周期**：6 周，所有必做项已完成 ✅  
> **最后更新**：2026年6月13日

## 八、数据流全景

```
用户浏览器 (http://localhost:80)
    │
    ▼
Nginx (Port 80)
    ├── /api/* → 反向代理到后端 :3000
    └── 其他   → SPA 静态文件 (frontend/dist/)
    │
    ▼
React SPA (Vite + TDesign + ECharts)
    │  REST API (axios + JWT Bearer token)
    ▼
Express Server (Node.js, Port 3000)
    ├── /api/user          → 用户认证（JWT 签发，7天有效期）+ 头像上传（multer）
    ├── /api/tasks         → 任务 CRUD（自动补全天气/学期阶段）
    ├── /api/analysis      → ⛓️ DeepSeek 归因分析（5种拖延类型+JSON容错）
    ├── /api/checkin       → 每日打卡 + 🏅徽章自动判定引擎（7种规则）+ 徽章规则含进度
    ├── /api/dashboard     → 数据聚合（Promise.all 并行查询，~50ms）
    └── /api/intervention  → ⛓️ DeepSeek 干预方案（CBT策略库+个性化建议）
    │
    ├── /uploads           → 静态文件服务（头像等上传资源，express.static）
    │
    ├── DeepSeek API (chatCompletion, 自动重试+30s超时)
    └── MySQL 8.0 (Prisma ORM, Port 3307)
```

### 核心业务闭环

```
记录拖延 → DeepSeek AI 归因分析（5种类型判断）
    → 可视化画像（饼图+热力图+趋势线）
    → 个性化干预建议（CBT 3步行动方案）
    → 每日打卡养成习惯
    → 徽章激励正向循环（7种徽章自动判定）
```

#### 测试设计要点

**Mock 策略：**
- Prisma Client：通过 `vi.mock('../src/config/database')` 全局 mock，所有数据库操作返回可控数据
- DeepSeek API：通过 `vi.mock('../src/services/llmService')` mock，返回预设 JSON 响应
- Logger：通过 `vi.mock('../src/utils/logger')` 静默，避免测试输出噪音

**数据工厂模式：**
- 使用 `buildMockXxx()` 函数生成测试数据，支持 `overrides` 参数局部覆盖
- 无需手动构造完整数据对象，减少测试样板代码

**时间冻结：**
- 打卡模块使用 `vi.useFakeTimers()` + `vi.setSystemTime()` 冻结时间
- 确保连续天数计算可预测且可重复
